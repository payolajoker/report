# Codex 작업 지시서: 서울 동남부 도시 연담화 문서 개선

## 0. 대상 파일 및 컨텍스트

작업 대상: `서울-동남부-도시-연담화-세부사례-근거정리.md`

이 보고서는 서울 동남부 연담화를 6개 사례 축(미사, 감일, 교산, 위례, 동부 확장, 영동대로)으로 나눠 분석한 문서다. 전반적인 논리 구조와 근거는 견고하지만, 헤더 레벨 오류, 사례 구조 불균형, 서술 위치 문제, 철도 명칭 부정확 등 다수의 편집·구조 개선이 필요하다.

**반드시 위에서 아래 순서대로 작업하라.** 각 Task 완료 후 커밋하라: `Task N: <간단 요약>` 형식.

**원칙**: 주장·논리·각주·출처 내용은 수정하지 않는다. 구조·형식·명칭 정확성·누락 항목 보완만 한다.

## Task 1: 헤더 레벨 오류 수정 (가장 우선)

### 문제

`## 4. 세부 사례 분류` (h2) 다음에 `## 4-1. 사례 A` ~ `## 4-6. 사례 F` 가 모두 h2로 작성되어 있다. 4-X는 4의 하위 항목이므로 h3이 되어야 한다. 이에 따라 그 아래 `### 성격`, `### 왜 별도 사례인가`, `### 공간 배치와 지도 판독`, `### 연담화 해석 포인트`, `### 주요 근거 출처`도 h4로 내려가야 한다.

### 수정 내용

아래 6개 사례 헤더를 `##` → `###` 으로 변경한다.

| 변경 전 | 변경 후 |
|---|---|
| `## 4-1. 사례 A: 강동-미사-하남도심 축` | `### 4-1. 사례 A: 강동-미사-하남도심 축` |
| `## 4-2. 사례 B: 송파-감일 축` | `### 4-2. 사례 B: 송파-감일 축` |
| `## 4-3. 사례 C: 송파-교산-하남도심 축` | `### 4-3. 사례 C: 송파-교산-하남도심 축` |
| `## 4-4. 사례 D: 송파 거여·마천-위례-복정·남위례 축` | `### 4-4. 사례 D: 송파 거여·마천-위례-복정·남위례 축` |
| `## 4-5. 사례 E: 강동-하남-구리·남양주 동부 확장 축` | `### 4-5. 사례 E: 강동-하남-구리·남양주 동부 확장 축` |
| `## 4-6. 사례 F: 강남 삼성·영동대로 환승핵` | `### 4-6. 사례 F: 강남 삼성·영동대로 환승핵` |

이와 동시에, 각 사례 내부의 소항목 헤더도 `###` → `####` 으로 변경한다. 해당 소항목은 다음과 같다 (사례별로 동일 패턴):

- `### 성격` → `#### 성격`
- `### 왜 별도 사례인가` → `#### 왜 별도 사례인가`
- `### 공간 배치와 지도 판독` → `#### 공간 배치와 지도 판독`
- `### 연담화 해석 포인트` → `#### 연담화 해석 포인트`
- `### 주요 근거 출처` → `#### 주요 근거 출처`
- `### 하남측 위례를 따로 봐야 하는 이유` → `#### 하남측 위례를 따로 봐야 하는 이유`
- `### 성남측 세부 판독: 복정·남위례·판교` → `#### 성남측 세부 판독: 복정·남위례·판교`

§5의 소항목도 동일하게 처리한다:

- `### 5-1. 서울 접속 방향이 다르다` → 이건 `## 5. 사례 간 비교`의 하위이므로 `### 5-1.`이 맞다. **변경하지 않는다.**

### 확인 방법

- `grep "^## 4-" 서울-동남부-도시-연담화-세부사례-근거정리.md` 결과가 0건이어야 한다.
- `grep "^### 4-" 서울-동남부-도시-연담화-세부사례-근거정리.md` 결과가 6건이어야 한다.
- `grep "^### 성격" 서울-동남부-도시-연담화-세부사례-근거정리.md` 결과가 0건 (모두 ####로 바뀜).
- 내용(텍스트)은 한 글자도 바뀌지 않아야 한다.

## Task 2: 사례 F에 `공간 배치와 지도 판독` 소항목 추가

### 문제

사례 A~E는 모두 `#### 공간 배치와 지도 판독` 소항목(Task 1 완료 후 기준)이 있다. 사례 F(강남 삼성·영동대로 환승핵)에만 이 항목이 없어 구조가 불균형하다. 사례 F는 주거지가 아닌 환승·업무 결절점이라는 특성상 지리적 배치 서술이 짧더라도 항목 자체는 있어야 한다.

### 수정 내용

Task 1 완료 후 기준으로, 사례 F의 `#### 왜 별도 사례인가` 항목과 `#### 연담화 해석 포인트` 항목 사이에 다음 소항목을 삽입한다.

삽입 위치: `#### 왜 별도 사례인가` 블록의 마지막 줄(`- 즉 서울 동남부 연담화의 교통 흐름은 단순히 서울 경계에서 끝나지 않고, 강남 핵심 기능지로 수렴한다.`) 바로 다음.

삽입 내용:

```markdown
#### 공간 배치와 지도 판독

삼성역·봉은사역 일대 영동대로는 강남구의 중심 업무축에서도 가장 고밀도 환승 기능이 집중된 구간이다. 공간적으로 보면 이 일대는 동측의 잠실·송파와 서측의 서초·반포 사이에서 수도권 남부 철도망의 수렴점 역할을 한다. 북측의 GTX-A/C 지하 환승구조와 남측의 버스환승 체계가 지하-지상 양 방향으로 쌓이는 구조이며, 이를 국제교류복합지구(MICE) 개발 계획이 감싸고 있다. 따라서 영동대로 환승핵은 단순한 역세권이 아니라, 수도권 동남부에서 유입되는 통근·업무 흐름이 최종적으로 수렴하는 고차 도시 기능 거점으로 읽어야 한다.[^7][^8][^9]
```

### 확인 방법

- 사례 F에 `#### 공간 배치와 지도 판독` 항목이 존재하는가.
- 기존 `#### 왜 별도 사례인가`와 `#### 연담화 해석 포인트` 사이에 위치하는가.
- 삽입된 텍스트가 기존 사례 F 서술과 모순되지 않는가.

## Task 3: 철도 명칭 부정확 수정

### 문제

사례 B(감일) 절의 `#### 연담화 해석 포인트` 항목에 다음 문장이 있다:

> 감일은 서울과 하남 사이의 완충지대가 아니라, 서울 주거 수요가 직접 밀려 들어간 **접경형 확장지**로 볼 수 있다. 미사가 강동과의 기존 광역철도 연결을 바탕으로 성장했다면, 감일은 **송파 인접성 + 향후 3호선 연장축**이라는 기대 위에서 재편되는 공간이다.

"3호선 연장축"이라는 표현이 부정확하다. 경기도 공식 자료(기본계획 승인 보도자료, gnews.gg.go.kr)는 송파하남선을 "오금역~하남시청역 11.7km" 독립 노선으로 제시하며, 3호선 연장이라고 명시하지 않는다. 이 문서 자체도 §5-2에서 "감일·교산은 **송파하남선** 중심의 미래 연결체계"라고 올바르게 표현하고 있어 내부 불일치가 있다.

### 수정 내용

해당 문장에서 "향후 3호선 연장축"을 "향후 송파하남선 축"으로 교체한다.

변경 전:
```
감일은 **송파 인접성 + 향후 3호선 연장축**이라는 기대 위에서 재편되는 공간이다.
```

변경 후:
```
감일은 **송파 인접성 + 향후 송파하남선 축**이라는 기대 위에서 재편되는 공간이다.
```

### 확인 방법

- `grep "3호선 연장" 서울-동남부-도시-연담화-세부사례-근거정리.md` 결과가 0건이어야 한다.
- 사례 C(교산) 절의 서울 접속 핵심 항목에 `송파구 오금역 방향` 및 `송파하남선(3호선 연장 성격)`이 남아 있다. 이 부분은 "3호선 연장 성격"이라는 표현이 괄호로 묶여 부연 설명으로 쓰인 것이라 맥락이 다르다. **이 부분은 수정하지 않는다.** (단, "3호선 연장 성격"이라는 표현이 맥락상 여전히 부정확하다고 판단되면, 각주를 달아 경기도 공식 자료는 독립 노선으로 제시한다는 점을 병기해도 된다 — 선택사항.)

## Task 4: 판교 서술 위치 조정

### 문제

§4-4(사례 D: 위례 절)의 `#### 성남측 세부 판독: 복정·남위례·판교` 소항목에서 판교를 다루고 있다. 그러나 판교는 위례신도시와 행정구역도 다르고(성남 분당구 vs 수정구), 생활권도 다르며, 이 문서가 설정한 6개 사례 축 어디에도 판교는 직접 포함되지 않는다. 판교를 위례 절 안에서 "비교 지점"으로 언급하면서 "위례와 동일한 경계 연속지대는 아니지만"이라고 스스로도 인정하고 있어, 서술 위치가 구조와 맞지 않는다.

### 수정 내용

판교 서술을 위례 절(§4-4) 에서 분리해 §5(사례 간 비교) 안의 적절한 위치에 재배치한다.

#### 4-1. 위례 절에서 판교 단락 제거

`#### 성남측 세부 판독: 복정·남위례·판교` 소항목에서 판교를 다루는 세 번째 단락("셋째, **판교**는 위례와 달리…"로 시작하는 단락 전체)을 삭제한다.

소항목 제목도 `#### 성남측 세부 판독: 복정·남위례·판교` → `#### 성남측 세부 판독: 복정·남위례`로 변경한다.

#### 4-2. §5에 판교 비교 항목 신설

`## 5. 사례 간 비교: 왜 하남을 하나로 보면 안 되는가` 맨 끝(§5-4 거버넌스 단락 이후)에 다음 항목을 추가한다.

```markdown
### 5-5. 보론: 성남 판교는 왜 별도 비교점으로 다루어야 하는가

성남은 위례(복정·남위례)를 통해 서울 동남부 연담화의 경계 결절에 직접 참여하지만, 판교는 그와 다른 위상을 가진다. 판교테크노밸리 일대는 서울 경계에 직접 맞닿은 연속 시가지가 아니라, 성남 내부에서 고차 업무·상업 기능을 담당하는 독립적 중심지다. 성남시 공식 자료는 8호선 판교연장이 판교테크노밸리와 주요 업무·상업시설 접근성을 높이고, 모란역-판교역 환승을 통해 수도권 동남부 철도 네트워크를 강화한다고 설명한다.[^18][^19] 따라서 판교는 이 문서의 6개 사례 축에는 포함되지 않지만, 성남이 서울 동남부 연담화 체계 속에서 단순 주거 배후지가 아니라 **고차 기능극**으로 작동한다는 점을 보여주는 비교 지점이다.
```

### 확인 방법

- `grep "판교" 서울-동남부-도시-연담화-세부사례-근거정리.md` 결과에서 §4-4 범위 내 판교 서술이 사라졌는가.
- §5-5 항목이 새로 생성됐는가.
- [^18][^19] 각주가 이미 존재하므로 중복 정의하지 않는다 — 확인할 것.
- 위례 절 내부의 복정·남위례 서술(첫째, 둘째 단락)은 그대로 남아 있어야 한다.

## Task 5: 초록 섹션 추가

### 문제

이 보고서를 제외한 다른 5개 최종 보고서는 모두 `## 초록` 섹션을 h2 레벨로 갖고 있다. 이 문서는 `## 1. 문서 목적`이 그 역할을 대신하고 있어 형식이 다르다. 독자가 문서 간 일관성을 기대할 때, 초록이 없으면 요지를 빠르게 파악하기 어렵다.

### 수정 내용

문서 제목(`# 서울 동남부 도시 연담화 세부 사례 및 근거 정리`) 바로 아래, `## 1. 문서 목적` 바로 위에 `## 초록` 섹션을 삽입한다.

삽입 내용:

```markdown
## 초록

이 글은 서울 동남부(강남·송파·강동)와 인접 도시 사이에서 진행되는 도시 연담화를 단일 현상이 아닌, 접속 방식과 도시 성격이 서로 다른 6개 사례 축의 중첩으로 읽는다. 분석 대상은 ①강동-미사-하남도심 축, ②송파-감일 축, ③송파-교산-하남도심 축, ④송파 거여·마천-위례-복정·남위례 축, ⑤강동-하남-구리·남양주 동부 확장 축, ⑥강남 삼성·영동대로 환승핵이다. 같은 하남권이라도 미사는 이미 개통된 철도 기반 확장지이고, 감일은 접경형 주거 확장지이며, 교산은 국가 주도 대규모 계획도시다. 위례는 서울·성남·하남에 걸친 행정분리-생활권통합형 신도시다. 강동-하남-남양주 동부축은 하남을 넘어 남양주 왕숙·별내까지 이어지는 장거리 외연 확장이며, 영동대로 환승핵은 이 외연들을 강남의 고차 중심지 기능으로 수렴시키는 결절점이다. 근거는 공공주택지구 고시(토지이음·EUM), LH 공개데이터, 대도시권 광역교통시행계획(대광위), 서울시·경기도·하남시·성남시 공식 자료에 한정한다.
```

### 확인 방법

- 문서 제목 다음, `## 1. 문서 목적` 이전에 `## 초록` 항목이 있는가.
- 초록 내용이 본문의 6개 사례 분류 및 핵심 판단과 일치하는가.
- `## 1. 문서 목적` 이하 본문은 한 글자도 바뀌지 않았는가.

## Task 6: 하남선 철도 명칭 표기 일관성 확인

### 문제

본문에서 하남선을 지칭할 때 "하남선(5호선 연장)"(§4-1 공간 배치 항목), "하남선"(§5-2), "5호선 연장"처럼 일관되지 않게 표기하는 경우가 있다. 첫 등장에서 "(5호선 연장)" 또는 "(하남선)" 병기를 한 번 하고, 이후에는 단일 명칭을 쓰는 것이 가독성에 좋다.

### 수정 내용

문서 전체에서 하남선 관련 표기를 점검한다.

- 처음 등장하는 곳(§4-1 서울 접속 핵심 항목: `철도 축: **하남선(5호선 연장)**`): 이미 올바르게 병기되어 있다. 유지.
- 이후 "하남선"으로만 표기된 곳: 유지 (첫 등장에서 병기했으므로 이후는 단독 명칭 허용).
- `대광위 운영현황상 하남선은 **상일동~검단산 7.7km, 2021년 개통** 광역철도로 제시된다.` 이 문장에서 "2021년 개통" 부분은 대광위 공식 자료가 "2021-03-27 개통"으로 제시한다. 일관성을 위해 `2021-03-27 개통`으로 표기한다.

변경 전:
```
대광위 운영현황상 하남선은 **상일동~검단산 7.7km, 2021년 개통** 광역철도로 제시된다.
```

변경 후:
```
대광위 운영현황상 하남선은 **상일동~검단산 7.7km, 2021-03-27 개통** 광역철도로 제시된다.
```

### 확인 방법

- "2021년 개통"이 "2021-03-27 개통"으로 변경됐는가.
- 다른 하남선 관련 표기는 변경 없는가.

## Task 7: 내부 링크 구조 점검 및 관련 문서 블록 위치 확인

### 문제

현재 문서의 `## 1. 문서 목적` 절 끝(첫 `---` 바로 직전)에 이미 `> **관련 문서**` 블록이 존재한다. 이 블록은 이 문서와 연관된 다른 보고서로의 링크를 제공한다. 위치는 적절하다.

한편 Task 1~6 완료 후 헤더 레벨이 바뀌면, 내부 앵커 링크(예: `§4-3`, `§4-4`처럼 섹션을 지목하는 링크)가 있다면 깨질 수 있다.

### 수정 내용

#### 7-1. 관련 문서 블록 위치 유지

기존 관련 문서 블록(`## 1. 문서 목적` 절 내, 첫 `---` 직전)을 그대로 유지한다. 위치나 내용을 변경하지 않는다.

#### 7-2. 내부 앵커 점검

다른 보고서에서 이 문서를 참조할 때 사용하는 앵커(`#4-3`, `#4-4` 등)가 Task 1에서 헤더 레벨이 바뀌어도 앵커 텍스트 자체는 동일하게 유지되는지 확인한다. GitHub Markdown에서 헤더 레벨(##/###)은 앵커 슬러그에 영향을 주지 않으므로, 앵커 값(`4-1-사례-a-...` 형식)은 변경되지 않는다. 별도 수정 불필요.

#### 7-3. 관련 문서 블록 링크 텍스트 보완

현재 관련 문서 블록의 링크 중 `[국가철도망 vs 도시철도망 비교](./국가철도망-구축계획과-도시철도망-구축계획-비교.md)` 항목에 간단한 설명이 없다. 다른 항목에는 모두 "- [제목](경로): 설명" 또는 "- [제목](경로) — 설명" 형태의 부연이 있다.

변경 전:
```markdown
> - 송파하남선·강동하남남양주선 등의 법정 계획 근거: [국가철도망 vs 도시철도망 비교](./국가철도망-구축계획과-도시철도망-구축계획-비교.md)
```

이미 설명이 "송파하남선·강동하남남양주선 등의 법정 계획 근거:"로 있으므로 **수정 불필요**. 확인 후 넘어간다.

### 확인 방법

- 관련 문서 블록이 `## 1. 문서 목적` 절 끝, 첫 `---` 직전에 그대로 있는가.
- 링크가 실제 존재하는 파일을 가리키는가.
