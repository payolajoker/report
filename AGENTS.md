# AGENTS - 연구 보고 아카이브

## 저장소 목적

이 저장소는 한국 수도권 공간구조, 신도시, 철도, 행정구역, 도시 기능 위계에 대한 개인 연구 아카이브다. LLM은 보고서 작성과 자료 정리를 돕지만, 모든 사실 진술은 공식 1차 자료와 원문 재구성 자료로 검증하는 것을 원칙으로 한다.

## 기본 진입 순서

1. `README.md` - 사람용 전체 소개, 공개 보고서, 원문 재구성 도서, 보조 자료의 큰 지도.
2. `index.md` - LLM 작업용 색인. 질문이나 수정 작업을 시작할 때 우선 확인한다.
3. `log.md` - 최근 ingest, query, update, lint 기록. 새 자료 반영이나 중요한 정리 작업 뒤에 append한다.
4. `REPORT_STYLE.md` - 보고서 작성 원칙, 자료 위계, 문체와 표/그림 규칙.
5. 관련 하위 폴더의 `AGENTS.md` - 주제별 별도 규칙이 있는 경우 반드시 함께 따른다.

## 지식 계층

- `sources/` 또는 원본 PDF 파일: 원천 자료. 가능한 한 수정하지 않는다. 원본 교체가 필요하면 이유와 출처를 `log.md`에 남긴다.
- `books/*/docs/`: PDF를 LLM과 사람이 읽기 쉽게 재구성한 Markdown. 원문 쪽수, 페이지 이미지, 표, 그림 설명, 주요 수치 인덱스를 보존한다.
- `books/*/dist/`: 긴 컨텍스트나 RAG 입력용 LLM 단일 번들. 본문 판단의 최종 근거가 아니라 탐색 편의용 산출물이다.
- 루트 `*.md`: 공개 보고서 또는 핵심 조사 메모. 독립적으로 읽히는 문서이며, 사실과 해석을 분리해야 한다.
- `supplements/`: 방법론, 데이터, 조사 메모, source anchor, 주제별 보조 자료.
- `assets/` 및 `supplements/*/images/`: 보고서와 설명을 보조하는 이미지. 출처와 용도를 캡션이나 README에 남긴다.
- `scripts/`: 지도, 도표, 데이터 산출 스크립트. 산출물의 재현 가능성을 해치지 않도록 입력 경로와 출력 경로를 명확히 둔다.

## 작업 방식

### Ingest

새 원문 자료를 반영할 때는 다음 순서를 따른다.

1. 원본을 적절한 `sources/` 위치에 보관한다.
2. 필요한 경우 `docs/`, `stats.md`, `manifest.yml`, `source-page-index.md`, `dist/`를 생성하거나 갱신한다.
3. 자료가 기존 보고서의 주장, 수치, 비교, 해석에 영향을 주는지 확인한다.
4. 영향을 받는 루트 보고서, supplements 문서, README 또는 `index.md`를 갱신한다.
5. 작업 결과와 미해결 쟁점을 `log.md`에 append한다.

### Query

질문에 답할 때는 `index.md`를 먼저 읽고, 관련 보고서와 원문 재구성 자료를 좁혀서 확인한다. 답변은 가능한 한 저장소 내 문서 경로와 원출처를 함께 근거로 삼는다. 좋은 비교표, 요약, 연결 발견은 필요하면 새 보고서나 보조 메모로 남길 수 있다.

### Update

기존 보고서를 수정할 때는 본문만 고치지 말고 관련 색인과 보조 자료도 함께 확인한다. 특히 다음 경우에는 `index.md`와 `log.md`를 갱신한다.

- 새 공개 보고서가 추가되거나 제목, 범위, 역할이 바뀐 경우.
- 새 원문 도서나 대형 PDF 재구성 자료가 추가된 경우.
- 기존 보고서의 핵심 결론, 수치, 출처 위계가 달라진 경우.
- 반복 재사용되는 주제나 엔티티가 새로 드러난 경우.

### Lint

정기적으로 다음 항목을 점검한다.

- README와 `index.md`의 링크가 실제 파일과 맞는지.
- 루트 보고서의 핵심 주장과 books/supplements의 원문 근거가 어긋나지 않는지.
- 최신 정책, 법령, 통계가 필요한 문장이 오래된 자료에만 기대고 있지 않은지.
- 같은 개념이 여러 문서에서 다른 이름으로 반복되지 않는지.
- 보고서에 쓰인 수치가 `stats.md`, `manifest.yml`, source anchor, 원문 쪽수와 연결되는지.

## 출처와 서술 원칙

- 확인된 사실과 해석을 분리한다.
- 법령, 고시, 공식 통계, 국토교통부/지자체/공공기관 원문, 동시대 신문 등 1차 자료를 우선한다.
- 기관 편찬 자료와 후대 해설은 보조 근거로 사용한다.
- 최신성이 중요한 정책, 법령, 통계, 사업 일정은 현재 자료를 다시 확인한 뒤 서술한다.
- 불확실한 내용은 단정하지 않고 `보강 필요`, `추정`, `불확실`, `정밀 대조 필요`처럼 상태를 드러낸다.
- 과거 지명은 가능한 한 "당시 지명 / 현재 지명" 형식으로 병기한다.
- 행정구역, 계획구역, 생활권, 기능권을 혼동하지 않는다.

## 편집 규칙

- 저장소 안의 링크는 repo 상대경로를 사용한다. 사용자 홈 디렉터리나 임시 작업 폴더의 절대경로를 문서에 남기지 않는다.
- PDF, JPG, PNG 등 큰 바이너리는 Git LFS 대상 규칙을 유지한다.
- 대량 페이지 이미지는 한 디렉터리에 몰지 않는다. 권별·장별 하위 디렉터리로 나눠 디렉터리 폭을 작게 유지한다.
- `.claude/`와 worktree 산출물은 작업 참고용일 뿐, 공개 아카이브의 일부로 취급하지 않는다.
- 하위 폴더에 별도 `AGENTS.md`가 있으면 루트 규칙보다 구체 규칙을 우선한다.
- 이미 존재하는 보고서의 문체와 자료 위계를 유지하고, 불필요한 전면 개편은 피한다.
