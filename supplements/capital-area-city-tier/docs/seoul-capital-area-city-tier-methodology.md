# 수도권 중국식 도시등급 대응 프레임워크 v1.0

- 작성일: 2026-04-13
- 작성자: OpenCode / Sisyphus
- 상태: 초안이지만 재판별 가능한 운영 문서
- 적용 범위: 서울특별시, 인천광역시, 경기도
- 목적: 중국식 `1선·신1선·2선...`과 유사한 도시기능 등급 체계를, 한국의 **공식 통계**만으로 수도권에 맞게 재구성하고 반복 판별 가능하게 만드는 것
- 현재 run-1 계산 범위: **수도권 66개 기초지자체만 포함**, 경기도 일반구 sub-layer는 일시 제외

## 1. 이 문서의 목적

이 문서는 단순한 설명문이 아니라, 아래 4가지를 동시에 만족하도록 만든 운영 문서다.

1. 중국식 도시등급 개념을 짧고 정확하게 설명한다.
2. 그 개념을 한국 수도권에 옮길 때 무엇을 그대로 쓰고 무엇을 바꿔야 하는지 근거를 남긴다.
3. 공식 지표가 바뀌거나 신규 지표가 추가되더라도 다시 판별할 수 있도록 절차와 점수 규칙을 남긴다.
4. 현재 시점 기준의 권장 단위와 1~9등급 배정안을 남긴다.

---

## 2. 중국식 도시등급이란 무엇인가

중국에서 흔히 말하는 `1선도시, 신1선도시, 2선도시...`는 국가의 단일 법정 표준이라기보다, 실제로는 **민간·미디어형 도시 위계 체계**에 가깝다. 특히 최근 대중적으로 가장 영향력 있는 체계는 **第一财经·新一线城市研究所**의 도시 매력 랭킹이다.

이 체계가 보려는 것은 단순한 행정지위가 아니라 대체로 다음과 같은 요소다.

- 상업 자원 집적도
- 도시 허브성
- 도시인의 활동성
- 신경제 경쟁력
- 미래 잠재력

즉 중국식 도시등급은 본질적으로 **도시의 시장성, 중심성, 소비력, 산업 성장성, 장래성**을 함께 본다.

---

## 3. 왜 한국에서도 비슷한 프레임을 만들 수 있는가

한국은 중국처럼 도시를 수백 개 두고 순위를 매기는 구조는 아니지만, 수도권 내부만 놓고 보면 실제로는 다음과 같은 비교 수요가 있다.

- 어느 지역이 수도권 핵심 업무중심지인가
- 어느 지역이 상급 주거지이자 인구 흡입지인가
- 어느 지역이 산업 거점인가
- 어느 지역이 자족성이 약한 외곽 주거지인가
- 어느 지역이 군 단위의 주변부인가

즉 한국 수도권에도 `핵심 중심지 / 강한 부중심 / 성숙 주거지 / 외곽 혼합지역 / 주변부` 같은 위계는 분명히 존재한다. 다만 이를 평가할 때는 중국 Yicai 체계를 그대로 베끼지 말고, **한국의 공식 통계 구조에 맞게 재설계**해야 한다.

## 4. 중국식 프레임과 비교해 한국형 지표를 고른 배경

### 4.1 중국식 핵심 개념과 한국형 대응

| 중국식 개념 | 한국형 대응 개념 | 이유 |
|---|---|---|
| 商业资源集聚度 | 사업체 밀도, 종사자 밀도, 중심상권·업무중심지 기능 | 한국에는 Yicai식 브랜드 데이터가 공식 통계로 통일되어 있지 않기 때문 |
| 城市枢纽性 | 통근 흡입력, 거주지내 취업 비중, 교통 접근성, 수도권 내 중심성 | 한국 수도권은 철도·통근권 구조가 도시 기능을 결정하는 비중이 매우 큼 |
| 城市人活跃度 | 인구 유입, 청년 유입, 취업 활력, 생활인구·이동 데이터 보조 활용 | 중국식 소비·야간활력 데이터를 한국에서 완전 동일하게 공식화하기 어려움 |
| 新经济竞争力 | 산업구조, 사업체수, 창업기업수, 제조업·서비스업 기반, 특허·고부가 산업 지표 | 한국은 산업·기업 관련 공식 통계가 상대적으로 강함 |
| 未来可塑性 | 인구증감, 청년순이동, 주택수급, 교육·의료·재정기반 | 장기 성장 가능성은 결국 사람과 서비스 기반으로 나타남 |

### 4.2 왜 이 선택이 더 적절한가

중국 Yicai는 상업 브랜드, 플랫폼 데이터, 소비 트렌드 같은 민간 데이터 활용 비중이 높다. 반면 이 프로젝트의 목표는 **공식 통계만으로 반복 가능한 판단 체계**를 만드는 것이다. 따라서 한국에서는 다음 원칙이 필요하다.

1. 민간 상권·부동산 랭킹은 보조 참고만 하고 공식 판단축에서는 제외한다.
2. 지표는 가능한 한 **전국 단위 공식 기관이 같은 기준으로 생산한 것**을 우선한다.
3. 수도권 특성상 `교통 허브`, `통근 흡입력`, `주거 선호`, `자족 일자리`를 강하게 반영한다.

## 6. 비교 단위 설계

이 문서에서 가장 중요한 설계는 지표가 아니라 **단위**다.

### 6.1 왜 단위가 중요한가

서울의 자치구는 독립된 기초지자체다. 인천의 구·군도 마찬가지다. 하지만 경기도의 일반구는 대부분 **독립 기초지자체가 아니라 시 내부의 행정구역**이다. 따라서 `강남구`와 `분당구`, `수원시`를 같은 1개의 사다리에서 곧바로 비교하면 행정적 자율성과 재정단위가 달라진다.

### 6.2 권장 단위

이 프로젝트의 **주 단위(main universe)** 는 다음으로 한다.

- 서울 25개 자치구
- 인천 8개 구 + 2개 군
- 경기 28개 시 + 3개 군

즉, **수도권 기초지자체 66개**를 메인 비교군으로 둔다.

### 6.3 별도 보조 단위

경기도 일반구는 **보조 계층(sub-city layer)** 으로 따로 다룬다.

해당 일반구:
- 수원시: 장안구, 권선구, 팔달구, 영통구
- 성남시: 수정구, 중원구, 분당구
- 안양시: 만안구, 동안구
- 안산시: 상록구, 단원구
- 고양시: 덕양구, 일산동구, 일산서구
- 용인시: 처인구, 기흥구, 수지구
- 부천시: 원미구, 소사구, 오정구

이들은 메인 래더와 섞지 않고 별도 부속 래더로 평가한다.

다만 **run-1 실제 계산에서는 이 보조 계층을 제외**한다. 이유는 다음과 같다.

1. 메인 래더 66개 단위만으로도 현재 프로젝트 목적을 충족한다.
2. 일반구까지 동시에 넣으면 단위 혼선이 다시 생긴다.
3. 지금 단계의 우선순위는 `수도권 기초지자체 66개`의 공식 점수표를 먼저 완성하는 것이다.

## 7. 1~9등급 체계의 원칙

Oracle 검토 결과, 수도권이라는 상대적으로 좁은 공간에 1~9등급을 모두 공개하는 것은 **가짜 정밀도(false precision)** 위험이 있다. 그러나 사용자가 1~9 체계를 원하므로, 이 문서에서는 다음과 같이 운영한다.

### 7.1 운영 원칙

1. **내부 운영은 1~9 전체를 쓸 수 있다.**
2. 다만 실제 대외 공개 시에는 1, 3, 5, 7, 9의 5개 실질 밴드를 우선 권장한다.
3. 2, 4, 6, 8은 경계 사례(borderline cases)에만 사용한다.
4. 매년 또는 지표갱신 시 재판별할 때, 3년 평균 또는 최근 3개 시점 평균을 우선 사용한다.

### 7.2 등급 의미

| 등급 | 의미 |
|---|---|
| 1 | 수도권 최상위 핵심 기능 중심지 |
| 2 | 최상위에 준하는 핵심 상위 중심지 |
| 3 | 강한 상위 중심지 |
| 4 | 상위권이지만 핵심성은 다소 약한 부중심 |
| 5 | 성숙한 중상위 생활·업무 중심지 |
| 6 | 중간권 주거·산업·생활 중심지 |
| 7 | 외곽형 또는 기능 편중형 지역 |
| 8 | 주변부 군·약한 외곽 도시 |
| 9 | 수도권 내 구조적으로 가장 주변적인 군/도서 지역 |

### 8.3 차원 3: 고용과 노동시장 강도

- 이유: KOSIS 자체 안내상 지역 간 상대적 물가 수준 비교에 직접 쓰기 부적절한 경우가 있다.

### 9.3 민간 부동산 랭킹
- 이유: 반복 가능성과 공공성 약함

### 9.4 공항·항만 특수효과만 반영한 고용 총량

## 10. 재판별 가능한 운영 규칙

이 절은 나중에 지표가 바뀌거나 새 데이터가 나왔을 때 다시 돌릴 수 있도록 만든 절차다.

### 10.1 기본 운영 주기
- 권장 갱신 주기: 연 1회
- 권장 기준 시점: 전년도 확정치 또는 최근 3개 시점 평균

### 10.2 비교군 설정

메인 래더는 아래 세 peer set으로 나눈다.

1. 서울·인천의 자치구
2. 경기도 시
3. 수도권 군(인천 군 + 경기 군)

이유는 동일한 행정 구조와 기능 규모를 가진 집단 내에서 먼저 상대 위치를 잡아야 총량 왜곡을 줄일 수 있기 때문이다.

### 10.3 표준화 방식

각 지표는 아래 순서로 처리한다.

1. 지표 방향 통일
   - 높을수록 좋은 지표는 그대로 사용
   - 낮을수록 좋은 지표는 역방향 변환
2. 규모 보정
   - 가능한 경우 인구당, 취업자당, 면적당으로 변환
3. 이상치 점검
   - 공항, 항만, 중앙업무지구, 도서지역은 플래그 처리
4. 3개년 평균
   - 단년도 급등락 완화
5. peer set 내부 백분위화
   - 같은 구조의 지역끼리 비교

### 10.4 차원 점수 계산

각 차원은 0~100점으로 환산한다.

- 차원 내 세부지표가 3개 이상이면 평균 사용
- 특정 지표가 결측이면 같은 차원 내 다른 공식 대체지표 사용
- 대체지표가 없으면 차원 총가중치 내에서 재배분하지 말고 `결측 플래그`를 남김

### 10.5-B 통근 축의 최종 운영 원칙

통근 축은 본 프로젝트의 병목이었고, 최종적으로 아래처럼 고정한다.

- **1차 대표지표(primary)**: `DT_1PA2020`의 `T40 주간 인구 지수`
- **2차 보조지표(secondary)**:
  - `T11 유입 인구-통근`
  - `T21 유출 인구-통근`

이유는 다음과 같다.

1. 수도권에서는 단순 거주인구보다 **낮/업무 중심성**이 지역 기능을 더 잘 드러낸다.

3. 동일 census family 내부에서 유입과 유출을 함께 볼 수 있어 구조적으로 해석이 가능하다.

운영 규칙:

- 통근 축은 **구조지표(structural indicator)** 로 간주한다.
- 인구총조사 기반이므로 매년 바뀌는 지표로 보지 않는다.
- census가 갱신되기 전까지는 마지막 확정치를 유지한다.
- 대신 고용, 인구이동, 재정 같은 annual/monthly 지표를 갱신해 전체 점수 변화를 반영한다.

- 한계: 66개 시군구 전체를 한 번에 당겨오는 exact nationwide family는 이번 단계에서 확정하지 못함

1. 메인 래더 계산 시 사업체 차원은 `partial` 플래그를 부여한다.

5. **폐지 지표 처리**
   - 동일 개념 대체지표로 교체
   - 대체 불가 시 해당 차원 내부에서 다른 지표 평균으로 유지

### 13.1 현재 통합 baseline 상태

2026-04-13 기준으로 아래 run-1 통합 baseline 파일이 생성되었다.

- `../data/seoul-capital-area-run1-merged-baseline.csv`

이 파일은 아래 3개 실데이터 파일을 `(region, unit_name)` 기준으로 결합한 것이다.

- population: `../data/seoul-capital-area-population-baseline-2026-03.csv`
- migration: `../data/seoul-capital-area-migration-baseline-2026-02.csv`
- employment: `../data/seoul-capital-area-employment-baseline-2025-H2.csv`

현재 이 통합 파일에서 보장되는 것은 다음과 같다.

1. run-1 main universe 66개 단위 전부 포함
2. 인구, 이동, 고용 3축 모두 결합 완료
3. migration 누락 0
4. employment 누락 0

아직 남아 있는 것은 보조차원 데이터를 같은 방식으로 붙이는 작업이다.

### 13.2 재정 축 run-1 확정 경로

재정 축은 run-1에서 제외하지 않고, 아래 KOSIS e-지방지표 지역 데이터 엔드포인트로 유지한다.

- 재정자립도
  - `unitySrvcId=955`
  - `stdIdctId=381`
- 재정자주도
  - `unitySrvcId=1106`
  - `stdIdctId=382`
- 호출 엔드포인트
  - `POST https://kosis.kr/visual/eRegionIndex/selectYearToRegionData.do`

2025년, `clsfLevel=2` 기준으로 서울 25구, 인천 10구군, 경기 31시군의 66개 단위 값을 직접 반환하는 것이 확인되었고, run-1 fiscal baseline 파일로 저장되었다.

- `../data/seoul-capital-area-fiscal-baseline-2025.csv`

### 13.3 통근 축 run-1 확정 경로

통근 축은 아래 공식 표를 사용해 run-1에서 유지한다.

- KOSIS `DT_1PA2020`
- 항목: `T40` 주간 인구 지수
- 기준 시점: `2020`

실제 구현에서는 KOSIS `statHtml`의 center frame tree에서 하위 시군구 66개를 선택한 뒤, `defaultClassArr` 를 최하위 코드로 고정하고 재조회하여 값을 확보했다. baseline 파일은 아래에 저장되었다.

- `../data/seoul-capital-area-commuting-baseline-2020.csv`

run-1에서는 통근 축의 primary indicator로 `T40` 만 우선 채택한다. `T11`/`T21`은 추후 보조지표로 확장 가능하다.

### 13.4 지가변동률 run-1 보강지표 확정 경로

지가변동률은 core source-of-record 축으로는 채택하지 않지만, **보강(supplementary) 지표**로는 채택한다.

- 공식 다운로드 경로:
  - `https://www.data.go.kr/data/15063473/fileData.do`
  - direct CSV content URL:
    - `https://www.data.go.kr/cmm/cmm/fileDownload.do?atchFileId=FILE_000000003207353&fileDetailSn=1&insertDataPrcus=N`
- 데이터셋명:
  - `국토교통부_읍면동 지가변동률`

run-1 적용 원칙은 아래와 같다.

1. **보강지표로만 사용**한다.
2. REB source-of-record 값과 완전 동일한 대표계열로 보지 않는다.
3. data.go 공개 페이지의 공식 경고대로, 시스템 사용 용도에 맞춰 가공된 값일 수 있음을 명시한다.

run-1에서는 2025년 `평균지가변동률` 중 **최신 비영(非零) 월 값**을 선택해 66개 단위 보강 baseline으로 저장했다.

- `../data/seoul-capital-area-land-price-baseline-2025-latest.csv`

## 15. 최종 결론

이 프로젝트는 중국식 도시등급의 핵심 철학, 즉 **중심성·시장성·생활기반·성장잠재력**을 한국 수도권에 맞게 옮긴 것이다. 다만 중국식 민간 상업랭킹을 그대로 모사하지 않고, **한국의 공식 지표 체계로 반복 가능하게 재구성했다**는 점이 핵심 차이다.

가장 중요한 운영 원칙은 세 가지다.

1. **단위를 잘못 섞지 말 것**
2. **공식 통계만 쓸 것**
3. **1~9등급은 쓰되, 실제 해석은 밴드 중심으로 할 것**

이 원칙만 지키면, 향후 지표가 바뀌더라도 이 문서만으로 다시 판별할 수 있다.
