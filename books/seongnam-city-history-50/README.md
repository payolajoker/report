## 자료 성격

이 자료군은 단일 역사책이라기보다 성남을 주제별로 분해한 도시사 데이터셋에 가깝다. 자연환경, 인구·토지이용, 옛 지도, 교통·장시, 전근대 역사, 광주대단지, 시 승격과 도시개발, 자치·산업·교육·복지·문화, 공간구조, 본시가지·분당·판교·위례·고등·신촌의 권역별 변천, 생활사와 사전류가 별도 권으로 나뉘어 있다.

## LLM 가공 계획

### 1단계: 원천 보존과 권별 인벤토리 확정

- `sources/manifest.json`을 기준으로 원 게시글 URL, 원 파일명, 저장 파일명, 권차, 용량, 쪽수를 보존한다.
- PDF는 GitHub에 올릴 때 반드시 Git LFS 대상으로 둔다.
- 각 권의 앞부분 목차를 추출해 `docs/seongnam-city-history-50/source-page-index.md`의 권별 목차 색인으로 옮긴다.

### 2단계: 권별 Markdown 변환

- OCR은 기본적으로 사용하지 않는다. 현재 샘플 확인상 대부분 텍스트 레이어가 있으므로 `pdftotext -layout` 기반으로 1차 추출한다.
- 각 권은 `docs/seongnam-city-history-50/vol-XX-<slug>.md`로 나누고, 모든 섹션에 원본 쪽수 표기를 붙인다.
- 표는 Markdown 표로 정리하되, 깨지는 복합 표는 fenced block과 원문 페이지 이미지 링크를 함께 둔다.

### 3단계: 지도·사진·도판 보존

- 제3권, 제8권, 제29-38권, 제48권은 도판 가치가 높으므로 주요 페이지를 `assets/pages/vol-XX/`와 `assets/figures/`로 별도 렌더링한다.
- 전체 페이지 이미지는 단일 디렉터리에 몰지 않고 `assets/pages/vol-01/`-`assets/pages/vol-50/` 권별 하위 디렉터리에 보관한다.
- 광주대단지, 분당, 판교, 위례, 본시가지, 상대원, 고등·신촌 등 공간사 도판은 본문 요약과 직접 연결한다.
- 도판 파일은 원문 권차와 페이지를 파일명에 넣어 감사 가능하게 만든다.

### 4단계: LLM용 주제 색인

권차 중심 색인과 별도로 다음의 교차 색인을 만든다.

- `places.md`: 수정구, 중원구, 분당구, 광주대단지, 본시가지, 상대원, 분당, 판교, 위례, 대장, 고등·신촌 등 장소 색인
- `timeline.md`: 광주대단지, 시 승격, 분당 개발, 판교 개발, 위례 개발, 산업단지 변천 등 연표
- `events.md`: 8·10 사건, 시 승격, 주요 개발계획, 행정구역 변화
- `figures.md`: 지도·사진·계획도 목록과 원문 페이지
- `stats.md`: 인구, 토지이용, 산업, 통행, 주택, 학교, 복지 등 수치 표
- `relations.md`: 성남 원도심-분당-판교-위례-서울의 관계도

### 5단계: 단일 LLM 번들 생성

- `dist/seongnam-city-history-50.llm.md`를 생성한다.
- 본문 전체를 무리하게 한 파일에 밀어 넣지 않고, 먼저 `priority-urban-history.llm.md`를 만든다.
- 우선 번들은 제8권, 제9권, 제29권, 제30권, 제33-35권, 제37권을 중심으로 구성한다.
- 전체 50권 번들은 토큰 규모가 매우 크므로 권별 번들과 통합 색인을 분리한다.

### 6단계: 신도시 보고서와 연결

- `대한민국-신도시의-역사.md`의 성남·광주대단지·분당·판교·위례 절에 50년사 권차별 근거를 연결한다.
- 기존 40년사 인용은 보조자료로 낮추고, 50년사 제8권과 제9권을 우선 근거로 갱신한다.
- 성남은 "광주대단지-성남시 승격-분당-판교-위례"가 한 행정구역 안에 겹친 압축 사례로 별도 분석 노드를 만든다.
